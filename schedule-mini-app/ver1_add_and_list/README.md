# ver1_add_and_list

スケジュール管理アプリ（ミニ）のVer.1。予定をメモリ上に追加・一覧表示するだけの最小構成。
プログラムを終了すると内容は消える（ファイル保存はVer.2以降で対応予定）。

## プログラムについて

1件の予定を「年・月・日・予定内容」の4項目で構造体として保持し、配列に追加していく。
対応している操作は次の3つのみ。

- `1`：予定を追加する
- `2`：登録済みの予定を一覧表示する
- `0`：終了する

## 実行方法

```bash
gcc -Wall -Wextra -o schedule schedule.c
./schedule
```
## 使い方

実行するとメニューが表示されるので、番号を入力する。

```
=== スケジュール管理アプリ (Ver.1) ===
1: 予定を追加
2: 予定一覧を表示
0: 終了
番号を入力してください:
```

### 1: 予定を追加する場合

年月日を入力し、続けて予定の内容を入力する。年月日は次の2通りの形式に対応している。

- 半角スペース区切り：`2026 8 1`
- 8桁連続表記：`20260801`

```
年月日を入力してください（例: 2026 8 1 または 20260801）: 2026 8 1
予定を入力してください: 夏休み計画
予定を追加しました。
```

```
年月日を入力してください（例: 2026 8 1 または 20260801）: 20260801
予定を入力してください: お盆休み
予定を追加しました。
```

どちらの形式で入力しても、一覧表示では `2026-08-01` の形式で表示される。

### 2: 一覧表示する場合

登録した順番に、`年-月-日  予定` の形式で表示される。

```
--- 予定一覧 ---
1: 2026-08-01  夏休み計画
2: 2026-08-01  お盆休み
```

### 入力に関する注意事項

年月日は「半角スペース区切り」または「8桁連続表記」のいずれかの形式で入力すること。それ以外の形式（例: `2026-8-1` のようなハイフン区切りや、数字以外の文字を含む入力）の場合は、「入力形式が正しくありません」と表示され、その回の追加処理は中止される。再度メニューから `1` を選び、対応している形式で入力し直すこと。

なお `2026-08-01` のようなハイフン区切りは、内部的には「解析には成功するが月・日がマイナス値になる」という形で不正な入力として検出され、エラーメッセージが表示される（ハイフンを負の符号と誤認識するため）。
