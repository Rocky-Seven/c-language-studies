# v6_1_partial_notify

対応記事: [C言語で ミニ日記アプリを作ろう！（6.1）：Ver.6からVer.6.1へ「部分採用＋通知」方式で挙動を改善](https://my-studies.org/lets-make-a-mini-diary-app-using-c-06-01/)

## 背景：なぜ「（現在 100 文字）」と表示されたのに、実際の入力総量は100を超えたのか

Ver.6では、入力中のカウンタは「実際に採用した文字数」を表示する（改行はカウントしない）。上限直前の状態からさらに行を入力すると、上限（100）に達するまでの分だけその行の先頭から部分採用し、超過分は捨てる仕様であった。

そのため、ユーザーが合計103文字入力していても、採用分は100文字ちょうどなので表示が100/100になる（保存も100文字でトリミング）。**何文字捨てられたのかが画面上では分からない**という問題があった。

## 改善内容（Ver.6 → Ver.6.1）

入力行の取り込み時に、その行の超過文字数（`discard`）を計算し、追記直後に通知するようにした。

```c
size_t line_chars = count_characters(line);
size_t discard = 0;
if (cur_len + line_chars > 100) {
    discard = (cur_len + line_chars) - 100;
}
...
if (discard > 0) {
    printf("※ 100文字の上限により、この行の末尾 %zu 文字は保存されませんでした。\n", discard);
}
```

これにより、「100と表示されたのは"採用後の文字数"で、末尾のX文字は保存されなかった」ことが画面上で明確になる。

## Ver.6との差分まとめ

| 項目 | Ver.6 | Ver.6.1 |
| --- | --- | --- |
| 上限到達時の挙動 | 超過分を無言で切り捨て | 超過分を切り捨てた上で「何文字捨てたか」を通知 |
| その他の機能・構造 | 変更なし | 変更なし |

## コンパイル・実行

```
gcc -std=c11 -D_POSIX_C_SOURCE=200809L -Wall -Wextra -O2 diary_v6_1.c -o diary_v6_1
./diary_v6_1
```

## 実行例

芭蕉『おくのほそ道』冒頭を入力した場合：

```
=== 今日の日記を入力してください（空行で終了） ===
月日は百代の過客にして、行きかふ年も、また旅人なり。
（現在 26 文字 / 最大100文字）
舟の上に生涯を浮かべ、馬の口にとらへて老いをむかふる者は、
（現在 55 文字 / 最大100文字）
日々旅にして旅をすみかとす。
（現在 69 文字 / 最大100文字）
古人も多く旅に死せるあり。
（現在 82 文字 / 最大100文字）
予もいづれかの年よりか、
（現在 94 文字 / 最大100文字）
片雲の風に誘はれて
（現在 100 文字 / 最大100文字）
※ 100文字の上限により、この行の末尾 3 文字は保存されませんでした。
```

動作確認済み（コンテナ環境では`LANG=C.utf8 LC_ALL=C.utf8`を指定して実行）。

## 補足

Ver.3〜Ver.5にあった「タイムスタンプ二重カッコ」のバグ、およびタイムゾーンの環境依存問題は、いずれもVer.6の時点ですでに解消済み（構造変更とTZ明示指定による）。したがって本バージョン（Ver.6.1）にも、GitHub限定の追加修正（`v4_1_fix`/`v5_1_fix`のような整備版）は不要である。
