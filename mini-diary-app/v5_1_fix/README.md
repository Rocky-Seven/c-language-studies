# v5_1_fix

**本バージョンに対応するブログ記事はありません。**Ver.5（[C言語で ミニ日記アプリを作ろう！（5）](https://my-studies.org/lets-make-a-mini-diary-app-using-c-05/)）をベースに、GitHubリポジトリ内でのみ管理する整備版として作成した。

ブログ公開（2025.08.11）から約1年が経過していることもあり、以降のバージョンで対応済みの2点の修正を、所要の整備としてVer.5に反映した。

## 修正点

### (1) タイムスタンプの二重カッコ（`]]`）を修正

`load_entries()` が `[YYYY-MM-DD（曜） HH:MM:SS]` 行を読み込む際、先頭の `[` のみを除去して末尾の `]` を除去していなかったため、表示時に閉じ括弧が二重になっていた（Ver.3〜Ver.5に共通）。

改行除去のあとに、末尾が `]` であれば取り除く処理を追加した（`v4_1_fix` と同内容）。

```c
// 改行を除去
line[strcspn(line, "\n")] = '\0';

// ★修正点：末尾の ']' を除去
size_t llen = strlen(line);
if (llen > 0 && line[llen - 1] == ']') {
    line[llen - 1] = '\0';
}

strncpy(temp.timestamp, line + 1, sizeof(temp.timestamp) - 1);
```

### (2) タイムゾーンをJST（Asia/Tokyo）に固定

`write_new_entry()` は `time()` / `localtime()` でタイムスタンプを生成しているため、実行環境のタイムゾーン設定に依存していた。GitHub Codespacesなど既定でUTCのコンテナ環境では、日本時間より9時間早い時刻が記録されてしまう。

`main()` の冒頭で明示的にタイムゾーンを `Asia/Tokyo` に設定することで、実行環境に依存せず常にJSTでタイムスタンプを記録するようにした（Ver.6で導入済みの手法と同内容）。

```c
int main(void) {
    setenv("TZ", "Asia/Tokyo", 1);
    tzset();
    ...
```

`setenv` / `tzset` の宣言を確実に可視化するため、ファイル冒頭に以下のマクロを追加している。

```c
#define _POSIX_C_SOURCE 200809L
```

## Ver.5 との差分まとめ

| 項目 | Ver.5 | Ver.5.1 |
| --- | --- | --- |
| 機能面 | 変更なし（メニュー／過去2件表示／新規入力／月次一覧） | 変更なし |
| タイムスタンプ表示の二重カッコ | あり（バグ） | 修正済み |
| タイムゾーン | 実行環境依存（UTCになりがち） | 常にJST（Asia/Tokyo）に固定 |
| 対応するブログ記事 | あり | なし（GitHub内のみの整備） |

## コンパイル・実行

Ver.6以降と同様、POSIX関数（`setenv`, `tzset`）の宣言を確実に可視化するため、以下のコマンドを推奨する。

```
gcc -std=c11 -D_POSIX_C_SOURCE=200809L -Wall -Wextra -O2 diary_v5_1.c -o diary_v5_1
./diary_v5_1
```

## 実行例

シェル側のタイムゾーンを `UTC` にした状態でも、アプリ内は正しくJSTで記録されることを確認済み。

```
【過去2件の記録】
[2026-07-14（火） 09:34:25]
整備テスト（TZ・カッコ修正確認）。
```

括弧が1つだけで、時刻もJST（日本時間）になっていることが確認できる。
